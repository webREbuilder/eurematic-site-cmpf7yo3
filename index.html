How Klara Works

Four steps from wearable to on-chain proof — zero sensitive data exposed

01

Wearable Connect

User authorizes Klara to read data from their wearable via OAuth. Raw biometrics never leave the user's device ecosystem.

OAuth 2.0 HealthKit WHOOP API
02

AI Analysis + DP Noise

Klara's ML models evaluate wellness achievements. Differential privacy (ε-DP) noise is applied, making individual inference mathematically impossible.

ε-DP Gaussian Noise Federated ML
03

ZK Proof Generation

A zero-knowledge proof is generated that cryptographically proves a threshold was met (e.g., "Active: YES") without revealing the underlying value.

Groth16 PLONK circom
04

On-Chain Attestation

The verified proof is posted to the smart contract. Protocols can instantly query attestations to unlock rewards, premiums, or research incentives.

EAS Solidity ERC-721
klara-sdk — quickstart.ts
3 Lines to Production
// Install: npm install @klara-protocol/sdk
import {'{ KlaraOracle }'} from '@klara-protocol/sdk';
const oracle = new KlaraOracle({'{ apiKey: process.env.KLARA_KEY }'});
// Request a privacy-preserving wellness attestation
const attestation = await oracle.requestAttestation({'{'}
userId: '0xUserWalletAddress',
achievement: 'ACTIVE_7_DAY_STREAK',
privacyBudget: 0.1, // epsilon value
chain: 'arbitrum-one'
{'}'});
// Returns: {'{ proof: "0x...", onChainTxHash: "0x...", piiExposed: false }'}
console.log(attestation.proof); // Ready for your smart contract ✓
